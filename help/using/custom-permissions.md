# 自訂權限 {#custom-permissions}

瞭解如何使用自訂許可權建立新的自訂許可權設定檔，並使用可設定的許可權來限制對Cloud Managers使用者程式、管道和環境的存取權。

>[!NOTE]
>
>此功能僅適用於[早期採用者計劃。](/help/release-notes/current.md#early-adoption)

## 簡介 {#introduction}

Cloud Manager有一組預先定義的角色，可控制對Cloud Manager各種功能的存取：

* 企業所有者
* 方案管理員
* 部署管理員
* 開發人員

自訂許可權可讓使用者使用可設定的許可權建立新的自訂許可權設定檔，以限制Cloud Manager使用者對程式、管道和環境的存取權。

>[!TIP]
>
>有關預先定義角色的詳細資訊，請參閱檔案 [角色型許可權。](/help/requirements/role-based-permissions.md)

## 使用自訂許可權 {#using}

若要建立和使用您自己的自訂許可權，需要三個步驟：

1. [建立新的產品設定檔。](#create)
1. [將自訂許可權指派給新的產品設定檔。](#assign-permissions)
1. [將使用者指派給新的產品設定檔。](#assign-users)

本節將詳細說明這些步驟。 您可能會發現參閱 [辭彙](#terms) 和 [可設定的許可權](#configurable-permissions) 區段建立自己的自訂許可權時。

>[!NOTE]
>
>您必須在Admin Console中擁有產品管理員許可權，才能為Cloud Manager建立新設定檔和管理許可權。

### 建立新的產品設定檔 {#create}

您必須先建立新的產品設定檔，才能指派自訂許可權。

1. 登入Cloud Manager於 [my.cloudmanager.adobe.com](https://my.cloudmanager.adobe.com/)

1. 選取產品 **AEM Managed Services**.

1. 搜尋名稱符合模式的和執行個體 `*-cloud-manager` 並點選或按一下以管理使用者和許可權。

1. 系統會將您重新導向至 **產品** Admin Console的索引標籤，您可以在其中管理cloud manager的使用者和許可權。 在Admin Console中，點選或按一下 **新設定檔** 按鈕。

![「新增設定檔」按鈕](/help/assets/admin-console-new-profile.png)

1. 提供設定檔的一般詳細資料。

   * **產品設定檔名稱**  — 設定檔的描述性名稱
   * **顯示名稱** - UI （選項）中顯示的縮寫名稱
   * **說明**  — 說明其用途的設定檔資訊性說明（選擇性）
   * **透過電子郵件通知使用者**  — 選取後，當使用者新增到此設定檔或從中移除時，將會收到電子郵件通知。

1. 點選或按一下 **儲存** 完成時。

新產品設定檔會儲存，並顯示在Admin Console的產品設定檔清單中。

### 指派自訂許可權給設定檔 {#assign-permissions}

現在您有了新的產品設定檔，您可以指派自訂許可權給它。

1. 在Admin Console中，點選或按一下 [您剛建立的新產品設定檔。](#create)

1. 在開啟的視窗中，選取 **許可權** 索引標籤以檢視可編輯許可權清單。

   ![可編輯的許可權](/help/assets/permissions-tab.png)

1. 點選或按一下 **編輯** 編輯許可權的連結。

1. 此 **編輯許可權** 視窗會開啟。
   * 您在上一步選取的許可權會在左欄中選取。
   * 可指派給許可權的許可權專案位於中間的欄，標籤為 **可用許可權** 個專案。
   * 指派的許可權專案位於右欄，並標示為 **包含的許可權專案**.

   ![編輯許可權專案](/help/assets/edit-permission-items.png)

1. 點選或按一下加號(`+`)圖示來將許可權專案新增至欄 **包含的許可權專案**.

   * 點選或按一下 `i` 圖示加以存取，以瞭解更多相關資訊。

1. 點選或按一下 **全部新增** 按鈕上方的 **可用許可權** 欄以新增所有許可權。 同樣點選或按一下 **全部移除** 以移除所有先前選取的許可權。

1. 點選或按一下 **儲存** 當您完成新產品設定檔的許可權專案定義時。

您的新產品設定檔現在會與其自訂許可權一起儲存。

### 將使用者指派給自訂許可權 {#assign-users}

您現在可以將使用者指派至您使用自訂許可權建立的新產品設定檔。

1. 在Admin Console中，點選或按一下 [您剛指派自訂許可權的新產品設定檔。](#assign-permissions)

1. 在開啟的視窗中，選取 **使用者** 標籤。

1. 點選或按一下 **新增使用者** 按鈕，並使用自訂許可權將使用者指派給您的新產品設定檔。

請參閱區段 **將使用者和使用者群組新增至產品設定檔** 檔案 [管理企業使用者的產品設定檔](https://helpx.adobe.com/enterprise/using/manage-product-profiles.html) 以取得有關如何使用Admin Console的詳細資訊。

## 可設定的許可權 {#configurable-permissions}

下列許可權可用於建立自訂設定檔。

| 權限 | 說明 |
|---|---|
| 計畫存取 | 允許使用者存取程式 |
| 計畫編輯 | 允許使用者編輯計畫 |
| 管道建立 | 允許使用者建立新管道 |
| 管道刪除 | 允許使用者刪除管道 |
| 管道編輯 | 允許使用者編輯管道 |
| 生產部署核准/拒絕 | 允許使用者核准或拒絕生產部署步驟 |
| 管道執行取消 | 允許使用者取消管道執行 |
| 管道執行開始 | 允許使用者開始新的管道執行 |
| 覆寫/拒絕重要量度失敗 | 允許使用者覆寫/拒絕重要量度失敗 |
| 生產部署排程 | 允許使用者排程生產部署步驟 |
| 存放庫資訊存取 | 允許使用者存取存放庫資訊並產生存取密碼 |

### 組織層級許可權 {#organization-level}

組織層級許可權是指組織中所有方案一律會授予的許可權。

下列許可權是組織層級的許可權：

* **存放庫資訊存取** 此租使用者/組織層級許可權可讓使用者產生使用者名稱、密碼和存放庫URL，以便存取和貢獻給客戶專案。
   * 組織中所有存放庫存取許可權的使用者名稱和密碼都是通用的，不過每個計畫的存放庫URL都是唯一的。
   * 請參閱檔案 [原始碼存放庫](/help/requirements/source-code-repository.md) 以取得詳細資訊。

## 辭彙 {#terms}

建立及管理自訂許可權和預先定義的角色時會用到下列術語。

| 術語 | 說明 |
|---|---|
| 預先定義的許可權 | 預先定義的角色，例如 **企業所有者**， **部署管理員**&#x200B;等 以控管Cloud Manager的各種功能。 有關預先定義角色的詳細資訊，請參閱檔案 [角色型許可權。](/help/requirements/role-based-permissions.md) |
| 自訂權限 | Cloud Manager功能可讓使用者建立許可權設定檔，以定義角色來控管Cloud Manager的支援功能 |
| 許可權設定檔 | 在Admin Console中建立，用於管理可設定的許可權，這些許可權將適用於屬於許可權設定檔的使用者 |
| 可設定的許可權 | 可在許可權設定檔中設定的Cloud Manager許可權 |
| 許可權專案 | 可套用許可權的程式、環境或管道資源 |

許可權專案是指將套用許可權的領域。 通常會是下列其中一項。

| 許可權專案型別 | 範例 | 說明 |
|---|---|---|
| 組織 | organization：companyA | 組織的所有適用資源。 資源可以是方案、環境或管道。 如果使用者為任何許可權新增組織，則該組織中的所有新資源也將具有該許可權。 |
| 計劃 | 方案A | 方案的所有適用資源 |
| 環境 | 方案A ：環境 | 適用於特定環境 |
| 管道 | 方案A ：管道 | 適用於特定管道 |

## 限制 {#limitations}

使用自訂許可權時，請牢記以下限制。

* A [可用的許可權集有限](#configurable-permissions) 用於建立自訂設定檔。
* 計畫、環境、管道等資源。 在Cloud Manager中建立可能需要兩分鐘的時間才能在Admin Console中顯示許可權設定。
* 在自訂許可權服務無法回應的罕見情況下，預先定義的設定檔仍然可用，並且預先定義的設定檔中的使用者仍然具有適當的存取權。

## 常見問題集 {#faq}

### 哪些許可權設定檔是預先定義的許可權設定檔？

* 企業所有者
* 方案管理員
* 部署管理員
* 開發人員

有關預先定義角色的詳細資訊，請參閱檔案 [角色型許可權。](/help/requirements/role-based-permissions.md)

### 預先定義的許可權設定檔與自訂設定檔的簡介有何不同？

預設產品設定檔和Cloud Manager角色繼續像之前一樣運作。

### 是否可以編輯預先定義的許可權設定檔？

否，預設設定檔為不可編輯。 您無法新增或移除預設許可權設定檔的許可權。 您只能在預先定義的設定檔中新增或移除使用者。

### 我是否應該刪除預先定義的許可權設定檔，因為自訂設定檔現在已可使用？

不得從Admin Console中刪除預先定義的許可權設定檔。

### 我可以新增使用者到多個許可權設定檔嗎？

可以，使用者可以屬於多個設定檔，包括預先定義和自訂許可權設定檔。 當使用者被指派到多個設定檔時，所有指派的許可權設定檔中的合併許可權將可供該使用者使用。

### 如果使用者有權編輯環境/管道，但無權存取包含環境/管道的程式，會發生什麼情況？

在此情況下，如果使用者沒有，他們將不能存取環境或管道 **計畫存取** 包含環境或管道的許可權。

### 如果我在同一IMS組織中同時有AEMas a Cloud Service和AMS計畫，會發生什麼情況？ 我是否可以從一個設定檔管理許可權？ {#ams-and-aemaacs}

您應該為每個產品型別建立個別的設定檔(亦即，一個用於AEM作為Cloud Service，另一個用於AdobeManaged Services或AMS)。
